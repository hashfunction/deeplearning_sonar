Building a Sonar with a Microphone and Deep Neural Networks¶

In this notebook, we'll try and see if we can determine the distance to a wall by emitting a sound with a particular frequency and listening to it by a microphone array.

Theoretically, the sample rate of the microphone is high enough to capture both, the original sound and its echo. We will record the sound emitted and its capture in a noisy room and then try and train a DNN to correlate the echo with the distance to the wall.

Data Collection¶

The microphone array used is a PS3 eye toy. The sound is emitted with a mono speaker placed behind the array. The sound and echo is recorded in 4 channel audio. The distance in a straight line is recorded between the microphone array and the wall in inches.

Methodology¶

We will use two microphones to train the network and then use the remaining two microphone recordings to test the robustness of the trained network.

def load_data(num_seconds, channels, sample_rate):
    sample_len = num_seconds * sample_rate
    
    data_x = np.empty((0, sample_len, channels))
    data_y = np.empty((0)) 
    for subdir, dirs, files in os.walk('data'):
        for file in files:
            filepath = subdir + os.sep + file
            distance = int(basename(normpath(subdir)))

            samples, data = scipy.io.wavfile.read(filepath)
            data = data[:sample_len]
            
            if (data.shape[0] < sample_len):
                continue
            
            if (len(data.shape) == 1):
                data = data.reshape(-1, 1)

            for channel in range(0, data.shape[1]):
                data_x = np.concatenate((data_x, data[:,channel].reshape(-1, sample_len, channels)), axis=0)
                data_y = np.append(data_y, distance)
    
    return (data_x, data_y)

def scale(X, x_min, x_max):
    nom = (X-X.min(axis=0))*(x_max-x_min)
    denom = X.max(axis=0) - X.min(axis=0)
    denom[denom==0] = 1
    return x_min + nom/denom

Data cleanup methodology¶

So looks like the audio files contain a sequence of chirps, followed by a gap and more sequence of chirps. Within each sequence, we can see that in the beginning there is a high decibal chirp, followed by ambient noise together with any echo's that might be present.

So, to clean up the input, lets ignore all the ambient noise in between the chirp sequences, and within each sequence lets separate out individual chirps.

def get_samples(data_x, data_y, width = 1300, cutoff = 0.5):
    next_sample_idx = np.argmax(data_x > cutoff)
    beep = next_sample_idx
    prev_beep = next_sample_idx
    #out_x = np.empty((0, width))
    out_x = np.empty((0, width))
    out_y = []

    while next_sample_idx > 0 :
        sample = data_x[beep:beep+width]
        if (len(sample) < width):
            break
            
        out_x = np.concatenate((out_x, sample.reshape(1, width)), axis=0)
        out_y.append(data_y)
        
        next_sample_idx = np.argmax(data_x[beep+width:] > cutoff)
        prev_beep = beep;
        beep += width + next_sample_idx
        
    return (out_x, np.array(out_y))

Further steps and investigations¶

From the analysis above, its clear that the data is not of very high quality and the S/N ratio is quite low. This is why its hard for us to visually distinguish some of the samples. Still we were able to achieve > 99% accuracy from our testing set.

Some next steps are outlined below.

  • Gather more data. This will give us a better gauge of the effectiveness of the model.
  • Look at the failed cases in more detail and see if we can spot the issues there.
  • Remove the original chirp from the data so we are feeding the network less noise.
  • Experiment with some blurring/averaging techniques to see if that can bring out the signal better.
  • Use a different mic. This was captured with a ps3 eye toy 4 microphone array with low fidelity.
  • Try different waveforms for the chirp to see if perhaps different frequencies yield better results.
  • I am also curious about how general this trained model is to variations in ambient noise.
